# Portal DREA v2.1.0-alpha.1 — Release Notes

**Data**: 2026-04-11
**Tipo**: alpha release — primeira entrega do Design System SGA
**Estado**: ready for internal SGA validation

---

## O que é novo

Portal COE e Portal SSCI passam a partilhar uma **fundação visual unificada** — o Design System SGA. A partir desta versão, os dois portais têm:

- **Shell Bar** light no topo da página com faixa institucional SGA de 4px azul (`#004C7B`) por baixo
- **Sidebar** light com navegação por grupos + indicador `border-left` para a secção activa
- **Splash screen** no arranque com logo SGA + tagline DREA (~1.5s)
- **Footer institucional** no fundo de cada página com versão + OACI + build date
- **Sistema de tokens** em 3 camadas (primitivos → semânticos → densidade) para cores, tipografia, espaçamento, animações
- **13 componentes** CSS partilhados (button, badge, card, form, table, tabs, dropdown, modal, toast, save-badge, skeleton, empty-state, stat-card)
- **Inter Variable** embebida no HTML como fonte primária para elementos `.ds-*` (legacy Segoe UI ainda domina o body durante a migração)
- **`icon-siren`** no sprite SVG para a emergency pill do Portal COE

### Impacto visual

**Portal COE**: a sidebar azul-escura e o header gradient desapareceram. O que fica: um chrome light, minimalista, com o 4px azul SGA a marcar a identidade. É a maior mudança visual da história do portal.

**Portal SSCI**: pela primeira vez tem um logo SGA visível. O gradient dark-blue header foi substituído pelo mesmo Shell Bar light do COE. Os 17 nav items do SSCI passam a usar `class="nav-btn"` (antes `.nav-item`). O contrato `openSection()` passa de 1 argumento para 2 (`openSection(id, event)`) — consistente com o COE. Density continua a ser `comfortable` (maior respiração que o COE `compact`).

### O que NÃO mudou

- **Funcionalidade operacional**: zero. Todos os fluxogramas, checklists, dashboards, fichas, contactos, secções, formulários funcionam exactamente como antes.
- **JavaScript contracts**: preservados. `openSection`, `updateHeaderClock` (COE), `updateClock` (SSCI), `cfgApplyToPortal`, `window.notify`, `window.awmAlert`, todos os selectors JS-load-bearing (`.awm-modal*`, `.awm-toast*`, `.uxp-savebadge*`, `.contact-card*`, `.alarm-card*`, `.cr-chip`, `.tab-content`, IDs `clockDisplay`, `headerUTC`, `headerDate`, `operatorBadge`, `psci-turno`, `psci-clock`) continuam a funcionar sem alterações.
- **Dados**: `localStorage` (`coe_awm_config`, `psci_awm_config`, `PSCI_CFG.contactos`, etc.) continuam inalterados. Nenhuma migração de dados necessária.
- **Print**: stylesheet `@media print` genérico + `@page` headers institucionais para ambos os portais. Legacy rules preservadas como defesa em profundidade.
- **Instalação**: continua a ser copiar o `.html` — single-file, offline-capable, zero dependencies.

## Breaking changes

Nenhum para o **utilizador final**. Para quem tem forks customizados:

- **Classe `.nav-item` no SSCI** foi renomeada para `.nav-btn`. Se tens CSS custom a estilizar `.nav-item` na sidebar do SSCI, precisas de actualizar para `.nav-btn`.
- **`openSection(id)`** passa a `openSection(id, event)` — se tens código JS custom que chama `openSection`, adiciona `event` como segundo argumento.
- **Tokens `--ds-*`** foram renomeados para `--*`. Se o teu CSS custom referencia `var(--ds-brand-primary)`, muda para `var(--brand-primary)`. Os tokens legacy (`--dark-blue`, `--medium-blue`, etc.) continuam disponíveis nos sources durante a migração (cleanup completo deferred).

## Novos portais futuros (Portal SOA, AVSEC, DIRECÇÃO)

Adicionar um novo portal a partir desta versão requer:

1. Criar `packages/portal-XXX/` com a estrutura padrão (`src/`, `scripts/build.py`, `portal.config.json`)
2. Escrever o HTML do portal consumindo o chrome DS:
   ```html
   <header class="ds-shell-bar">...</header>
   <aside class="ds-sidebar">...</aside>
   <main class="ds-page-grid">...</main>
   ```
3. Configurar `portal.config.json` com `density` (compact ou comfortable), `name`, `tagline`, `role`, `ui` feature flags
4. Correr `python scripts/build-all.py` — o build pipeline injecta automaticamente todos os tokens, fonts, sprite, componentes, chrome, e CSS de impressão

Nenhum novo ficheiro CSS é necessário no novo portal. Nenhuma duplicação de código. A mesma fundação serve todos.

## Known limitations (debt)

Itens descritos no CHANGELOG que ficam para versões futuras:

1. **Logo SGA real**: actualmente é um stub SVG com triângulo + texto. Substituir por vectorizado real quando disponível. Basta trocar `shared/assets/logo-sga-mark.svg` e `logo-sga-full.svg` — zero outras mudanças necessárias
2. **`ls-wrapper.js`, `esc-html.js`, `awm-modal.js`, `awm-toast.js`, `awm-save-badge.js`**: planeados mas não extraídos — SSCI ainda tem inline implementations. Deferred para Fase 2 ou uma minor release que faça esta extracção
3. **Body font-family**: continua a ser Segoe UI legacy. A troca para Inter pode ser feita em `shared/styles/base/typography.css` removendo o `:where()` scope, mas precisa de validação visual prévia
4. **Typography global switch**: actualmente só elementos dentro de `.ds-shell-bar`, `.ds-sidebar`, `.ds-page-grid`, `.ds-splash`, `.ds-footer` usam Inter. O resto da página (cards, forms, tabelas) continua em Segoe UI. A transição completa é deliberada — a fazer quando houver confiança na font licence e render consistency
5. **Namespace `.ds-*` em classes** (não tokens) ainda dispersa entre Plan 4 chrome classes e outras. Um cleanup pass pode padronizar
6. **Stats + dash cards** ainda usam a classe `stat-card` genérica do Plan 3 — SSCI/COE poderiam consumir esta mas continuam com as suas próprias variações (`.dash-card` em SSCI, `.occ-dash-card` em COE). Future unification
