# Portal DREA v2.2.0-alpha.1 — Release Notes

**Data**: 2026-04-12
**Tipo**: alpha release — Visual Polish do Design System SGA
**Estado**: ready for internal SGA validation

---

## O que é novo

O Design System SGA (fundação v2.1.0) é agora **aplicado aos conteúdos operacionais**. O resultado visual é uma transformação significativa:

### Dashboard SSCI

- 6 KPIs em **2 secções row-grouped** ("Estado operacional SCI" + "Performance e verificação")
- Cada KPI em **stat-card-gauge** com número grande em JetBrains Mono + barra de progresso horizontal
- **Estados activos**: OK (verde), Unknown (border tracejado + cinza), Critical (pulse vermelho animado + tint rosa)
- Indicador "actualizado há Xs" ao vivo

### Formulário de Ocorrência COE

- Layout **two-column**: form à esquerda, **sticky timer card** à direita
- Cronómetro grande começa a contar quando Hora Notificação é preenchida
- Summary rows (Occ, Notif, Voo, Aeronave) actualizam em tempo real
- Auto-save indicator
- Campos numéricos em JetBrains Mono (tabular numerals)

### Sidebar (ambos os portais)

- Nav items como **pill active SGA**: fundo sólido #004C7B, texto branco, shadow subtil
- Hover state cinza-azul claro

### Typography

- **JetBrains Mono** embebida (Regular + Bold, ~183 KB, OFL license)
- Todos os números, IDs, timestamps, coordenadas renderizam em monospace
- Inter Variable continua para texto UI

### Novos tokens e componentes

- 53 novos tokens DS (elevation, status quad, motion, pill, gauge, mono font)
- 6 novos componentes CSS + 2 modifiers
- Acessibilidade: pulse → badge fallback com prefers-reduced-motion, role=alert em critical states

## O que NÃO mudou

- Funcionalidade operacional: zero alterações
- Todos os fluxogramas, checklists, fichas, contactos funcionam igual
- JavaScript contracts preservados (openSection, notify, awmAlert, lsGet/lsSet, etc.)
- Dados localStorage inalterados
- Print: logo SGA no topo (integrado pre-Plan-7)
